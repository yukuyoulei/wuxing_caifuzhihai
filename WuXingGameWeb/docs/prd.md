# 五行元素冒险游戏需求文档\n
## 1. 游戏概述
\n### 1.1 游戏名称
五行探险记

### 1.2 游戏描述
一款基于五行相克原理的冒险游戏，玩家通过探险收集金木水火土五种元素，使用阴阳货币进行战斗辅助，在随机生成的地图中不断冒险前进。

### 1.3 技术架构
- 前端：HTML + JavaScript
- 后端：ASP.NET Core\n
## 2. 游戏玩法

### 2.1 初始设定
- 玩家出生位置：坐标(x:0, y:0)
- 初始资源：金木水火土五种元素各0个，阴阳货币各0个
- 初始技能：金木水火土五种基础技能各1级
- 出生点NPC：可补充元素至10个（任何元素低于10时触发）\n
### 2.2 探险机制
\n#### 2.2.1 移动规则
- 玩家选择出发探险后，系统随机四个方向之一：左上、左下、右上、右下
- 每次前进在当前方向中再次随机：
  - 随机到'左'：x坐标-1\n  - 随机到'右'：x坐标+1
  - 随机到'上'：y坐标-1
  - 随机到'下'：y坐标+1
\n#### 2.2.2 战斗系统
- 到达新位置后，根据距离(0,0)点的距离随机生成一定数量的五行元素
- 战斗流程：
  1. 玩家指定自己的一种元素\n  2. 依次点击对方元素，亮明具体类型和数量
  3. 元素克制判定：
     - 玩家元素克制对方：对方数量减半（受技能阳等级影响）
     - 玩家元素被克制：对方数量翻倍（受技能阴等级影响）
     - 其他情况：对方数量不变
  4. 数量对比：玩家元素数量 > 对方元素数量则本局胜利，否则失败
  5. 五种元素全部比对完毕后结算

#### 2.2.3 冒险结算
- 胜利条件：5局对比中胜利数量 >= 3
- 冒险成功：玩家从胜利的局次中选择一局，获取该局数量的元素
- 冒险失败：玩家返回(0,0)点

### 2.3 货币系统\n
#### 2.3.1 货币获取\n- 每次冒险成功后，根据当前位置距离(0,0)点的距离获得随机数量的阴或阳货币
- 每次只能获得一种货币
- 有概率不获得货币
\n#### 2.3.2 货币使用
- 冒险失败时可使用
- 消耗阴或阳货币强行改变一局对比结果
- 消耗数量 = 玩家当局小于对方的元素数量
- 每次冒险失败只能扭转一局
- 扭转后重新检查冒险是否成功

### 2.4 技能系统\n
#### 2.4.1 技能类型\n- 玩家初始拥有5个基础技能，分别对应金、木、水、火、土五种元素
- 每个技能初始等级为1级

#### 2.4.2 技能升级机制
- 升级消耗：
  - 对应五行元素（消耗数量较多）
  - 阴或阳货币（消耗数量较少）\n- 消耗数量随技能等级提升而增加

#### 2.4.3 技能效果
- 阴属性升级：提升该技能元素被克制时的减免效果，降低对方元素翻倍倍率
- 阳属性升级：提升该技能元素克制对方时的克制效果，增加对方元素削减比例
- 每个技能的阴阳属性可独立升级

## 3. 五行相克规则
- 金克木
- 木克土
- 土克水
- 水克火
- 火克金

## 4. 设计风格
- 配色方案：采用中国传统五行配色，金(金黄)、木(青绿)、水(深蓝)、火(赤红)、土(土黄)为主色调，背景使用深色调以突出元素色彩
- 视觉细节：元素图标采用圆形设计，带有柔和光晕效果，战斗时有克制关系的元素间显示连线动画，技能图标带有阴阳标识区分升级方向；每次场景切换时使用淡入淡出过渡动画，持续时间约0.3-0.5秒，营造流畅的游戏体验
- 整体布局：采用卡片式布局，左侧显示玩家状态(坐标、元素数量、货币、技能等级)，中央为战斗区域，右侧为操作按钮区
